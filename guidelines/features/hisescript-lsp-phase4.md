# HISEScript LSP — Phase 4 Implementation Plan

## Overview

Phase 4 builds a native C++ LSP proxy binary that bridges AI coding agents (Claude Code, OpenCode, Crush) to HISE's `POST /api/diagnose_script` REST endpoint. The proxy is a stateless forwarder: it receives LSP notifications over stdio, calls HISE's REST API, and maps the response back to LSP `publishDiagnostics`.

**Key change from earlier plan:** The proxy is a compiled JUCE C++ binary (not Python), eliminating all runtime dependencies. It lives in a separate git repo added as a submodule to HISE, so precompiled binaries don't bloat the main repo and users can opt out.

### Prerequisites (all complete)

## Architecture

```
Agent (Claude Code / OpenCode) edits .js file
  → file saved to disk
  → textDocument/didSave notification → hise-lsp binary (native, stdio)
    → POST http://localhost:1900/api/diagnose_script { filePath }
    ← { success, diagnostics: [...] } or connection error or 404
    → textDocument/publishDiagnostics back to agent
Agent sees diagnostics inline
  → fixes issues, re-saves, diagnostics update automatically
  → when done: calls recompile via MCP tool (separate from LSP)
```

**Token cost model:** The LSP communication is handled internally by the agent's framework. The agent only sees compact diagnostic messages (a few lines each). The full file content transfer to HISE is invisible to the agent's token budget.

#### Section 1: JSON-RPC Framing (~50 lines)

```cpp
#include <JuceHeader.h>
using namespace juce;

// All logging goes to stderr (stdout is the LSP channel)
static void log(const String& msg)
{
    std::cerr << msg.toStdString() << std::endl;
}

// Read a single LSP message from stdin.
// Format: "Content-Length: N\r\n\r\n{...json...}"
// Returns empty var on EOF.
static var readMessage()
{
    // Read headers line by line until empty line
    // Parse Content-Length from headers
    // Read exactly N bytes of body
    // Return JSON::parse(body)
    
    String line;
    int contentLength = -1;
    
    // Read headers
    while (true)
    {
        int c = std::cin.get();
        if (c == EOF) return {};
        
        if (c == '\n')
        {
            line = line.trimEnd(); // remove \r if present
            
            if (line.isEmpty())
                break; // empty line = end of headers
            
            if (line.startsWith("Content-Length:"))
                contentLength = line.fromFirstOccurrenceOf(":", false, false).trim().getIntValue();
            
            line.clear();
        }
        else
        {
            line += (char)c;
        }
    }
    
    if (contentLength <= 0) return {};
    
    // Read body
    std::vector<char> buffer(contentLength);
    std::cin.read(buffer.data(), contentLength);
    if (std::cin.gcount() != contentLength) return {};
    
    return JSON::parse(String(buffer.data(), (size_t)contentLength));
}

// Write a JSON-RPC message to stdout.
static void writeMessage(const var& msg)
{
    auto json = JSON::toString(msg);
    auto utf8 = json.toStdString();
    
    std::cout << "Content-Length: " << utf8.size() << "\r\n\r\n" << utf8;
    std::cout.flush();
}
```

#### Section 2: HISE REST Client (~40 lines)

```cpp
struct HiseConnection
{
    String host = "localhost";
    int port = 1900;
    
    // Result of a diagnose call
    struct Result
    {
        bool connected = false;     // false = connection refused
        bool success = false;       // from JSON response
        String errorMessage;        // from JSON response or connection error
        var diagnostics;            // the diagnostics array from HISE
    };
    
    Result diagnoseFile(const String& filePath) const
    {
        DynamicObject::Ptr body = new DynamicObject();
        body->setProperty("filePath", filePath);
        
        auto url = URL("http://" + host + ":" + String(port) + "/api/diagnose_script")
            .withPOSTData(JSON::toString(var(body.get())));
        
        URL::InputStreamOptions options(URL::ParameterHandling::inAddress);
        
        auto stream = url.createInputStream(
            options.withExtraHeaders("Content-Type: application/json")
                   .withConnectionTimeoutMs(5000));
        
        if (stream == nullptr)
            return { false, false, "Cannot connect to HISE runtime (is HISE running on port " 
                     + String(port) + "?)", {} };
        
        auto response = stream->readEntireStreamAsString();
        auto json = JSON::parse(response);
        
        if (!json.isObject())
            return { true, false, "Invalid response from HISE", {} };
        
        bool ok = (bool)json["success"];
        
        if (!ok)
            return { true, false, json["errorMessage"].toString(), {} };
        
        return { true, true, {}, json["diagnostics"] };
    }
};
```

**Important:** The `withConnectionTimeoutMs(5000)` prevents the binary from hanging indefinitely if HISE isn't running. 5 seconds is generous — localhost connections either succeed instantly or fail with connection refused.

#### Section 3: Diagnostic Mapping (~40 lines)

```cpp
// Map HISE severity string to LSP DiagnosticSeverity integer
// LSP spec: 1=Error, 2=Warning, 3=Information, 4=Hint
static int mapSeverity(const String& severity)
{
    if (severity == "error")   return 1;
    if (severity == "warning") return 2;
    if (severity == "info")    return 3;
    if (severity == "hint")    return 4;
    return 1; // default to error
}

// Convert a single HISE diagnostic to an LSP Diagnostic object
static var makeLspDiagnostic(int line, int column, int severity,
                              const String& message, const String& source,
                              const var& suggestions)
{
    // LSP lines/columns are 0-based, HISE lines are 1-based
    int lspLine = jmax(0, line - 1);
    int lspCol = jmax(0, column - 1);
    
    DynamicObject::Ptr range = new DynamicObject();
    DynamicObject::Ptr start = new DynamicObject();
    DynamicObject::Ptr end = new DynamicObject();
    start->setProperty("line", lspLine);
    start->setProperty("character", lspCol);
    end->setProperty("line", lspLine);
    end->setProperty("character", lspCol);
    range->setProperty("start", var(start.get()));
    range->setProperty("end", var(end.get()));
    
    DynamicObject::Ptr diag = new DynamicObject();
    diag->setProperty("range", var(range.get()));
    diag->setProperty("severity", severity);
    diag->setProperty("source", source.isNotEmpty() ? source : String("hisescript"));
    diag->setProperty("message", message);
    
    // Include suggestions in diagnostic data (agents can use this)
    if (suggestions.isArray() && suggestions.size() > 0)
    {
        DynamicObject::Ptr data = new DynamicObject();
        data->setProperty("suggestions", suggestions);
        diag->setProperty("data", var(data.get()));
    }
    
    return var(diag.get());
}

// Build LSP publishDiagnostics params from HISE response
static var mapHiseDiagnostics(const var& hiseDiagnostics, const String& uri)
{
    Array<var> lspDiags;
    
    if (hiseDiagnostics.isArray())
    {
        for (int i = 0; i < hiseDiagnostics.size(); i++)
        {
            auto& d = hiseDiagnostics[i];
            lspDiags.add(makeLspDiagnostic(
                (int)d["line"],
                (int)d["column"],
                mapSeverity(d["severity"].toString()),
                d["message"].toString(),
                d["source"].toString(),
                d["suggestions"]
            ));
        }
    }
    
    DynamicObject::Ptr params = new DynamicObject();
    params->setProperty("uri", uri);
    params->setProperty("diagnostics", var(lspDiags));
    return var(params.get());
}

// Create a synthetic error diagnostic at line 1 (for connection errors, 404s, etc.)
static var makeSyntheticError(const String& uri, const String& message)
{
    Array<var> diags;
    diags.add(makeLspDiagnostic(1, 1, 1 /*Error*/, message, "hisescript", {}));
    
    DynamicObject::Ptr params = new DynamicObject();
    params->setProperty("uri", uri);
    params->setProperty("diagnostics", var(diags));
    return var(params.get());
}
```

#### Section 4: LSP Handlers (~120 lines)

```cpp
class LspServer
{
public:
    HiseConnection hise;
    
    // Send a JSON-RPC notification (no id)
    void sendNotification(const String& method, const var& params)
    {
        DynamicObject::Ptr msg = new DynamicObject();
        msg->setProperty("jsonrpc", "2.0");
        msg->setProperty("method", method);
        msg->setProperty("params", params);
        writeMessage(var(msg.get()));
    }
    
    // Send a JSON-RPC response (with id)
    void sendResponse(const var& id, const var& result)
    {
        DynamicObject::Ptr msg = new DynamicObject();
        msg->setProperty("jsonrpc", "2.0");
        msg->setProperty("id", id);
        msg->setProperty("result", result);
        writeMessage(var(msg.get()));
    }
    
    // Publish diagnostics for a document
    void publishDiagnostics(const var& params)
    {
        sendNotification("textDocument/publishDiagnostics", params);
    }
    
    // Diagnose a file and publish results
    void diagnoseAndPublish(const String& uri)
    {
        // Convert file:// URI to local path
        auto filePath = uriToPath(uri);
        
        log("Diagnosing: " + filePath);
        
        auto result = hise.diagnoseFile(filePath);
        
        if (!result.connected)
        {
            // HISE not running — publish error diagnostic
            publishDiagnostics(makeSyntheticError(uri, result.errorMessage));
            return;
        }
        
        if (!result.success)
        {
            // HISE returned an error (404 = not included, etc.)
            publishDiagnostics(makeSyntheticError(uri, result.errorMessage));
            return;
        }
        
        // Map HISE diagnostics to LSP format and publish
        publishDiagnostics(mapHiseDiagnostics(result.diagnostics, uri));
    }
    
    // --- Handler methods ---
    
    var handleInitialize(const var& /*params*/)
    {
        DynamicObject::Ptr capabilities = new DynamicObject();
        
        // Text document sync: open/close + save notifications, no didChange
        DynamicObject::Ptr textDocSync = new DynamicObject();
        textDocSync->setProperty("openClose", true);
        textDocSync->setProperty("change", 0);  // 0 = None (no didChange)
        textDocSync->setProperty("save", true);
        capabilities->setProperty("textDocumentSync", var(textDocSync.get()));
        
        DynamicObject::Ptr result = new DynamicObject();
        result->setProperty("capabilities", var(capabilities.get()));
        
        // Server info
        DynamicObject::Ptr serverInfo = new DynamicObject();
        serverInfo->setProperty("name", "hise-lsp");
        serverInfo->setProperty("version", "1.0.0");
        result->setProperty("serverInfo", var(serverInfo.get()));
        
        return var(result.get());
    }
    
    void handleDidOpen(const var& params)
    {
        auto uri = params["textDocument"]["uri"].toString();
        if (uri.isNotEmpty())
            diagnoseAndPublish(uri);
    }
    
    void handleDidSave(const var& params)
    {
        auto uri = params["textDocument"]["uri"].toString();
        if (uri.isNotEmpty())
            diagnoseAndPublish(uri);
    }
    
    void handleDidClose(const var& params)
    {
        auto uri = params["textDocument"]["uri"].toString();
        if (uri.isNotEmpty())
        {
            // Clear diagnostics for closed file
            DynamicObject::Ptr clearParams = new DynamicObject();
            clearParams->setProperty("uri", uri);
            clearParams->setProperty("diagnostics", var(Array<var>()));
            publishDiagnostics(var(clearParams.get()));
        }
    }
    
    // --- URI helpers ---
    
    // Convert file:///path/to/file.js → D:/path/to/file.js (or /path/to/file.js on Unix)
    static String uriToPath(const String& uri)
    {
        auto path = uri;
        
        if (path.startsWith("file:///"))
        {
            path = path.fromFirstOccurrenceOf("file:///", false, false);
            
            // On Windows: file:///C:/path → C:/path
            // On Unix: file:///path → /path
            #if ! JUCE_WINDOWS
            path = "/" + path;
            #endif
        }
        else if (path.startsWith("file://"))
        {
            path = path.fromFirstOccurrenceOf("file://", false, false);
        }
        
        // Decode percent-encoded characters (%20 → space, etc.)
        path = URL::removeEscapeChars(path);
        
        // Normalize separators
        return path.replace("\\", "/");
    }
};
```

#### Section 5: Dispatch Loop (~50 lines)

```cpp
void runLspLoop(LspServer& server)
{
    bool running = true;
    bool shutdownRequested = false;
    
    while (running)
    {
        auto msg = readMessage();
        
        if (msg.isVoid())
        {
            // EOF on stdin — exit
            running = false;
            break;
        }
        
        auto method = msg["method"].toString();
        auto id = msg["id"];           // var — present for requests, absent for notifications
        auto params = msg["params"];
        bool isRequest = !id.isVoid(); // requests have an id, notifications don't
        
        log("← " + method + (isRequest ? " [" + id.toString() + "]" : ""));
        
        if (method == "initialize")
        {
            sendResponse(id, server.handleInitialize(params));
        }
        else if (method == "initialized")
        {
            // No-op notification
        }
        else if (method == "textDocument/didOpen")
        {
            server.handleDidOpen(params);
        }
        else if (method == "textDocument/didSave")
        {
            server.handleDidSave(params);
        }
        else if (method == "textDocument/didChange")
        {
            // No-op — we only care about saves (file must be on disk for HISE)
        }
        else if (method == "textDocument/didClose")
        {
            server.handleDidClose(params);
        }
        else if (method == "shutdown")
        {
            shutdownRequested = true;
            server.sendResponse(id, {});
        }
        else if (method == "exit")
        {
            running = false;
        }
        else if (isRequest)
        {
            // Unknown request — respond with MethodNotFound error
            DynamicObject::Ptr error = new DynamicObject();
            error->setProperty("code", -32601);
            error->setProperty("message", "Method not found: " + method);
            
            DynamicObject::Ptr response = new DynamicObject();
            response->setProperty("jsonrpc", "2.0");
            response->setProperty("id", id);
            response->setProperty("error", var(error.get()));
            writeMessage(var(response.get()));
        }
        // Unknown notifications are silently ignored (per LSP spec)
    }
}
```

**Note:** `sendResponse` in the dispatch loop should call `server.sendResponse()`. The code above uses a free function for illustration — in the actual implementation, consolidate into the `LspServer` class or use the server instance consistently.

#### Section 6: Main (~20 lines)

```cpp
int main(int argc, char* argv[])
{
    LspServer server;
    
    // Parse command line args
    for (int i = 1; i < argc; i++)
    {
        String arg(argv[i]);
        
        if (arg == "--port" && i + 1 < argc)
            server.hise.port = String(argv[++i]).getIntValue();
        else if (arg == "--host" && i + 1 < argc)
            server.hise.host = String(argv[++i]);
    }
    
    // Also check environment variables (lower priority than args)
    auto envPort = SystemStats::getEnvironmentVariable("HISE_LSP_PORT", "");
    auto envHost = SystemStats::getEnvironmentVariable("HISE_LSP_HOST", "");
    
    if (envPort.isNotEmpty() && server.hise.port == 1900)
        server.hise.port = envPort.getIntValue();
    if (envHost.isNotEmpty() && server.hise.host == "localhost")
        server.hise.host = envHost;
    
    log("hise-lsp v1.0.0 starting (HISE at " + server.hise.host + ":" + String(server.hise.port) + ")");
    
    // Set stdin/stdout to binary mode on Windows (prevent \n → \r\n translation)
    #if JUCE_WINDOWS
    _setmode(_fileno(stdin), _O_BINARY);
    _setmode(_fileno(stdout), _O_BINARY);
    #endif
    
    runLspLoop(server);
    
    return 0;
}
```

**Critical Windows detail:** `_setmode(_fileno(stdin), _O_BINARY)` and same for stdout. Without this, Windows translates `\n` to `\r\n` in stdout, corrupting the `Content-Length` framing. This requires `#include <io.h>` and `#include <fcntl.h>` on Windows.

## Diagnostic Behavior Table

| Scenario | LSP Response |
|----------|-------------|
| HISE running, file included, errors found | `publishDiagnostics` with mapped HISE diagnostics (line, col, severity, message, suggestions) |
| HISE running, file included, clean file | `publishDiagnostics` with empty diagnostics array (clears previous) |
| HISE running, file NOT included (404) | `publishDiagnostics` with single Error at line 1: `"This file is not included in any HISE script processor. Add include(\"filename\") and compile first."` |
| HISE not running (connection refused) | `publishDiagnostics` with single Error at line 1: `"Cannot connect to HISE runtime (is HISE running on port 1900?)"` |
| HISE returns unexpected error | `publishDiagnostics` with single Error at line 1: the error message from HISE |
| File opened (`didOpen`) | Diagnose immediately (same as didSave) |
| File saved (`didSave`) | Diagnose immediately |
| File changed (`didChange`) | No-op (file not on disk yet) |
| File closed (`didClose`) | Clear diagnostics (empty array) |

## Agent Configuration Reference

### Claude Code

Install the plugin at project scope:

```bash
claude --plugin-dir tools/hise_lsp_server/plugin
```

The plugin's `.lsp.json` maps `.js` files to the `hisescript` language and points to the precompiled binary. Claude Code spawns the binary automatically and pipes LSP messages over stdio.

**Per-project override:** The plugin is scoped to the project. Other projects (non-HISE) are not affected. If needed, disable the built-in TypeScript LSP for this project to avoid duplicate `.js` diagnostics (not usually necessary — both LSPs can coexist).

### OpenCode

Add to the project's `opencode.json`:

```json
{
  "lsp": {
    "hisescript": {
      "command": ["tools/hise_lsp_server/bin/windows/hise-lsp.exe"],
      "extensions": [".js"]
    }
  }
}
```

OpenCode's per-project config (`opencode.json` in project root) takes precedence over global config. The HISE LSP only activates in this project.

To disable the built-in TypeScript LSP (optional — avoids duplicate `.js` diagnostics):

```json
{
  "lsp": {
    "typescript": { "disabled": true },
    "hisescript": {
      "command": ["tools/hise_lsp_server/bin/windows/hise-lsp.exe"],
      "extensions": [".js"]
    }
  }
}
```

### Crush

Add to the project's `.crush.json` or `crush.json`:

```json
{
  "lsp": {
    "hisescript": {
      "command": "tools/hise_lsp_server/bin/windows/hise-lsp.exe",
      "args": []
    }
  }
}
```

### Platform paths

Replace `bin/windows/hise-lsp.exe` with the correct platform binary:

| Platform | Path |
|----------|------|
| Windows | `tools/hise_lsp_server/bin/windows/hise-lsp.exe` |
| macOS | `tools/hise_lsp_server/bin/macos/hise-lsp` |
| Linux | `tools/hise_lsp_server/bin/linux/hise-lsp` |

### Custom HISE port

If HISE is running on a non-default port:

```json
{
  "lsp": {
    "hisescript": {
      "command": ["tools/hise_lsp_server/bin/windows/hise-lsp.exe", "--port", "2000"],
      "extensions": [".js"]
    }
  }
}
```

Or set the environment variable `HISE_LSP_PORT=2000`.
