# OpenAPI Migration Guide

This guide explains how to migrate endpoint documentation from `rest-api.md` into structured `RouteMetadata` so it appears in the OpenAPI spec generated by `GET /`.

## ABSOLUTE RULES

1. **ASCII ONLY.** All text written into RouteMetadata strings (summary, description, parameter descriptions, variant descriptions, examples) MUST be ASCII-only. No em-dashes, no smart quotes, no ellipsis characters, no non-ASCII unicode of any kind. Use regular dashes (-), straight quotes, and three dots (...) instead. This applies to every string literal in RestApiDefinitions.cpp and any documentation files.

2. **One static function per endpoint.** Each migrated endpoint MUST be a static function in the `RestApiEndpoints` struct in `hi_backend/backend/ai_tools/RestApiDefinitions.cpp`. The function takes `Array<RouteMetadata>& m` and adds exactly one route. See the reference implementation `applyBuilder()` below.

## File structure

All route metadata lives in `hi_backend/backend/ai_tools/RestApiDefinitions.cpp`:

```cpp
struct RestApiEndpoints
{
    using RouteParameter = RestHelpers::RouteParameter;
    using ParamType      = RestHelpers::ParamType;
    using RouteMetadata  = RestHelpers::RouteMetadata;
    using ApiRoute       = RestHelpers::ApiRoute;

    // Each migrated endpoint gets its own static function
    static void applyBuilder(Array<RouteMetadata>& m) { /* ... */ }
    static void injectMidi(Array<RouteMetadata>& m)   { /* ... */ }
    // ...
};
```

The `getRouteMetadata()` function at the bottom of the file calls each helper in enum order. Unmigrated endpoints remain inline in `getRouteMetadata()`. As each endpoint is migrated, its inline entry is replaced by a call to its static helper.

## Reference implementation: applyBuilder()

This is the gold standard. Every migrated endpoint MUST follow this pattern:

```cpp
static void applyBuilder(Array<RouteMetadata>& m)
{
    // 1. Define complex sub-schemas first (discriminated unions, nested objects)
    auto opItem = RouteParameter(RestApiIds::op, "Operation object")
        .withType(ParamType::Object)
        .withDiscriminator("op")
        .withVariant("add", "Add a new module (type, parent, chain, name)")
        .withVariant("remove", "Remove a module (target)")
        // ... all variants ...
        .withProperty(RouteParameter(RestApiIds::op, "Operation type")
            .withEnumValues({ "add","remove","clone","set_attributes",... }))
        .withProperty(RouteParameter(RestApiIds::target, "Target module name").asOptional())
        // ... all properties across all variants ...
        ;

    auto diffEntry = RouteParameter(Identifier("entry"), "Diff entry")
        .withType(ParamType::Object)
        .withProperty(...)
        ;

    // 2. Build the route metadata with all fields
    m.add(RouteMetadata(ApiRoute::BuilderApply, "api/builder/apply")
        .withMethod(RestServer::POST)
        .withCategory("builder")
        .withSummary("Apply a set of operations to the module tree")     // concise
        .withDescription("Apply one or more operations to the module "   // detailed
            "tree in a single batch. When called inside an undo group "
            "(after push_group), the diff accumulates...")
        .withReturns("Diff summary showing the net effect of all operations")
        .withBodyParam(RouteParameter(RestApiIds::operations, "Non-empty array of operation objects")
            .withArrayItems(opItem))
        .withResponseField(RouteParameter(RestApiIds::scope, "Scope level")
            .withEnumValues({ "group", "root" }))
        .withResponseField(RouteParameter(RestApiIds::groupName, "Active group name"))
        .withResponseField(RouteParameter(RestApiIds::diff, "Array of diff entries")
            .withArrayItems(diffEntry))
        .withErrorCodes({ 400, 404, 409 })
        .withRequestExample(R"({"operations": [...]})")
        .withResponseExample(R"({"success": true, "result": {...}, "logs": [], "errors": []})"));
}
```

Key points:
- Complex sub-schemas (discriminated unions, nested objects) are built as local variables BEFORE the `m.add()` call
- `withSummary()` is a single sentence describing what the endpoint does
- `withDescription()` carries full behavioral detail
- All parameters have explicit types (not default String when they should be Int/Bool/etc.)
- Response fields describe what is inside `result`
- Error codes match the handler's `req->fail()` calls
- Request and response examples use `R"(...)"` raw string literals

## Step-by-step: Migrating an endpoint

### 1. Create the static function

Add a new static function to `RestApiEndpoints` in `RestApiDefinitions.cpp`:

```cpp
static void myEndpoint(Array<RouteMetadata>& m)
{
    // ... build metadata ...
}
```

### 2. Read the rest-api.md entry and the handler implementation

From rest-api.md, extract:
- A concise one-sentence summary
- Parameter names, types, required/optional, defaults
- Enum/allowed values
- Nested object structures
- Response fields inside `result`
- HTTP error status codes
- Request and response JSON examples
- Behavioral notes (fold into `description`)

From the handler in RestHelpers.cpp, verify:
- Which `req->fail()` status codes are used
- What fields the response DynamicObject contains
- Any conditional behavior worth documenting

### 3. Add parameter types

Before:
```cpp
.withBodyParam(RouteParameter(RestApiIds::moduleId, "Script processor ID"))
```

After:
```cpp
.withBodyParam(RouteParameter(RestApiIds::moduleId, "Script processor ID")
    .withExample("Interface"))
```

For typed parameters:
```cpp
.withBodyParam(RouteParameter(RestApiIds::durationMs, "Recording duration in ms (100-5000)")
    .withType(ParamType::Int)
    .withDefault("1000")
    .withExample("2000"))
```

For enums:
```cpp
.withBodyParam(RouteParameter(RestApiIds::mode, "Value interpretation mode")
    .withEnumValues({"value", "normalized", "raw"})
    .withDefault("value"))
```

### 4. Model nested objects

For a parameter that is a JSON object with known fields:

```cpp
.withBodyParam(RouteParameter(RestApiIds::callbacks, "Callback scripts")
    .withType(ParamType::Object)
    .withProperty(RouteParameter(Identifier("onInit"), "Init callback script").asOptional())
    .withProperty(RouteParameter(Identifier("onControl"), "Control callback script").asOptional()))
```

### 5. Model discriminated unions

For arrays of objects where a discriminator field determines the shape:

```cpp
auto itemSchema = RouteParameter(RestApiIds::op, "Operation object")
    .withType(ParamType::Object)
    .withDiscriminator("op")
    .withVariant("add", "Add a new module (type, parent, chain, name)")
    .withVariant("remove", "Remove a module (target)")
    // Add ALL possible child properties (union of all variants)
    .withProperty(RouteParameter(RestApiIds::op, "Operation type")
        .withEnumValues({"add", "remove"}))
    .withProperty(RouteParameter(RestApiIds::type, "Module type").asOptional())
    .withProperty(RouteParameter(RestApiIds::target, "Target module").asOptional());

.withBodyParam(RouteParameter(RestApiIds::operations, "Array of operations")
    .withArrayItems(itemSchema))
```

### 6. Add response fields

These describe the fields inside the `result` object of the response envelope:

```cpp
m.add(RouteMetadata(...)
    .withResponseField(RouteParameter(RestApiIds::scope, "Scope level")
        .withEnumValues({"group", "root"}))
    .withResponseField(RouteParameter(RestApiIds::diff, "Array of diff entries")
        .withArrayItems(diffEntrySchema)));
```

Without `withResponseField()`, the response schema shows `result` as a generic `{type: "object"}` with only the `returns` text as description.

### 7. Add error codes

```cpp
.withErrorCodes({400, 404, 409})
```

### 8. Add examples

```cpp
m.add(RouteMetadata(...)
    .withRequestExample(R"({"moduleId": "Interface", "expression": "Engine.getSampleRate()"})")
    .withResponseExample(R"({"success": true, "result": {"value": 44100}, "logs": [], "errors": []})"))
```

### 9. Write a summary

```cpp
.withSummary("Apply a set of operations to the module tree")
```

Single sentence. Describes what the endpoint does, not what it returns.

### 10. Fold behavioral notes into description

Before (in rest-api.md):
> **Notes**: When called inside an undo group, the diff accumulates all operations.

After:
```cpp
.withDescription("Apply operations to the module tree. "
    "When called inside an undo group (after push_group), "
    "the diff accumulates all operations in the group so far.")
```

### 11. Replace the inline entry in getRouteMetadata()

Replace the inline `m.add(...)` with a call to your new static function:

```cpp
// Before:
m.add(RouteMetadata(ApiRoute::BuilderApply, "api/builder/apply")
    .withMethod(RestServer::POST)
    .withCategory("builder")
    .withDescription("Apply one or more operations to the module tree in a single batch")
    .withBodyParam(...));

// After:
RestApiEndpoints::applyBuilder(m);
```

## ParamType reference

| ParamType | OpenAPI type | Use for |
|-----------|-------------|---------|
| `String` | `string` | Text, IDs, paths (default) |
| `Int` | `integer` | Counts, indices, MIDI numbers |
| `Float` | `number` | Velocities, frequencies, dB values |
| `Bool` | `boolean` | Flags, toggles |
| `Array` | `array` | Lists - use `withArrayItems()` for element schema |
| `Object` | `object` | Nested objects - use `withProperty()` for fields |
| `Enum` | `string` + `enum` | Fixed value sets - use `withEnumValues()` |

## Migration checklist per endpoint

For each entry in `rest-api.md`:

- [ ] Create a static function in `RestApiEndpoints` (named after the handler, e.g., `applyBuilder`, `injectMidi`)
- [ ] Add `withSummary()` - concise one-sentence description of what the endpoint does
- [ ] Add `ParamType` to all parameters (default String is often wrong for ints/bools)
- [ ] Add `withEnumValues()` where the docs list allowed values
- [ ] Add `withExample()` to key parameters
- [ ] Model nested objects with `withProperty()` / `withArrayItems()`
- [ ] Model discriminated unions with `withDiscriminator()` / `withVariant()`
- [ ] Add `withResponseField()` for all documented response fields
- [ ] Add `withErrorCodes()` from the error cases table
- [ ] Add `withRequestExample()` and `withResponseExample()` from the curl examples
- [ ] Fold behavioral notes into the `description` string
- [ ] Replace inline `m.add(...)` in `getRouteMetadata()` with `RestApiEndpoints::yourFunction(m);`
- [ ] Verify the OpenAPI output in editor.swagger.io
- [ ] Remove the endpoint's section from `rest-api.md` once fully migrated
